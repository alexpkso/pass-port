# Принципы дизайна Pass-Port

**Аудитория:** предприниматель с современным взглядом на дизайн.  
**Цель:** эффективность и удобство работы, а не эффектность.

---

## Главное

1. **Понятное представление информации** — таблицы, цифры и действия должны читаться быстро и без лишнего шума.
2. **Простота** — один чёткий путь к частым действиям; не перегружать экран декоративными элементами.
3. **Эффективность** — каждый элемент интерфейса должен помогать решать задачу (учёт, отчёты, решения).

## Что делаем

- **Типографика:** единая иерархия (Manrope), размеры по шкале (12 / 14 / 16 / 18 / 20–24 для заголовков).
- **Цвет:** фон, текст, один акцент (синий для действий), приглушённый текст для вторичной информации.
- **Пространство:** единая шкала отступов (4, 8, 12, 16, 24, 32 px); у таблиц и карточек — достаточные отступы.
- **Таблицы:** явные заголовки колонок, числа с выравниванием (tabular-nums), лёгкий hover по строке.
- **Действия:** основной CTA один и заметный; вторичные кнопки визуально слабее.
- **Анимация:** только с целью (обратная связь, наведение); короткие тайминги (150–300 ms). Пасхалки не должны мешать работе.

## Чего избегаем

- Декоративные градиенты и эффекты «ради красоты».
- Много анимаций на главном пути работы.
- Перегруженные карточки и пестрота цветов.
- Неочевидные или двусмысленные подписи кнопок и разделов.
