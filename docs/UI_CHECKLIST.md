# Чек-лист UI (на основе webd + фокус на эффективность)

Ориентир: предприниматель, современный взгляд, понятное и простое представление информации, удобство работы. **Цель — эффективность, а не эффектность.**

---

## 1. Перед началом (контекст и направление)

- [ ] **Цель интерфейса:** дашборд / учёт / отчёты — что пользователь должен делать в первую очередь?
- [ ] **Аудитория:** кто пользуется? (здесь: предприниматель, сам ведёт учёт)
- [ ] **Приоритет:** эффективность и ясность, а не «вау»-эффекты
- [ ] **Эстетика:** один основной стиль (для Pass-Port: чистый, минималистичный, с акцентом на контент)

---

## 2. Типографика

- [ ] **Иерархия:** заголовки / подзаголовки / тело / подписи — визуально различимы (размер, вес)
- [ ] **Размеры:** шкала без хаоса (например: 12px подписи, 14px тело, 16–18px подзаголовки, 20–24px заголовки)
- [ ] **Читаемость:** межбуквенное расстояние и межстрочный интервал комфортны для чтения таблиц и текста
- [ ] **Единый шрифт:** один основной шрифт (или пара: заголовки + тело) по всему проекту

---

## 3. Цвет и контраст

- [ ] **Фон и текст:** достаточный контраст (доступность, не «серое на сером»)
- [ ] **Акцент:** один цвет для действий (кнопки, ссылки) — не размывать множеством акцентов
- [ ] **Приглушённый текст:** вторичная информация (подписи, даты) — одним цветом muted
- [ ] **Без декора:** не добавлять градиенты/цвет «ради красоты», если они не несут смысла

---

## 4. Композиция и пространство

- [ ] **Сетка/выравнивание:** блоки и колонки выровнены, предсказуемая структура
- [ ] **Отступы:** единая шкала (например 4px: 4, 8, 12, 16, 24, 32) — не случайные числа
- [ ] **Воздух:** достаточно «пустого» места, чтобы не давить; таблицы и карточки не слипшиеся
- [ ] **Приоритет на экране:** главное (действия, ключевые цифры) заметно без поиска

---

## 5. Таблицы и данные

- [ ] **Заголовки колонок:** явно отделены от данных (фон, вес, размер)
- [ ] **Числа:** выровнены (например по правому краю), моноширинный шрифт для цифр при необходимости
- [ ] **Чередование строк / hover:** лёгкое выделение строки при наведении для удобства сканирования
- [ ] **Длинный текст:** обрезка/многоточие или перенос по правилам, без «разъезжающей» верстки

---

## 6. Действия и формы

- [ ] **Кнопки:** основной CTA один и понятный («Сохранить», «Выгрузить в CSV»), вторичные — визуально слабее
- [ ] **Формы:** поля с подписями или placeholder, логичный порядок; ошибки рядом с полем
- [ ] **Обратная связь:** загрузка (спиннер/состояние), успех/ошибка — явно и коротко

---

## 7. Движение и анимация

- [ ] **Только с целью:** анимация для обратной связи (открытие модалки, сохранение) или наведения, не «для красоты»
- [ ] **Коротко:** 150–300 ms для микровзаимодействий
- [ ] **Не мешать:** не отвлекать от чтения и ввода; пасхалки/декор — опционально и ненавязчиво

---

## 8. Доступность и удобство

- [ ] **Фокус:** видимый focus у кнопок, ссылок, полей ввода (для клавиатуры)
- [ ] **Подписи:** у иконок и кнопок есть текст или aria-label
- [ ] **Контраст:** текст и фон проходят по контрасту (WCAG 2.1 AA при необходимости)
- [ ] **Удобство работы:** минимум кликов до частых действий, понятные названия разделов

---

## 9. Производительность и ясность кода

- [ ] **Скорость:** страницы и списки открываются без долгой загрузки; тяжёлые блоки — лениво или по запросу
- [ ] **Понятная структура:** компоненты и стили названы так, чтобы было ясно, за что они отвечают

---

## 10. Проверка «эффективность, а не эффектность»

- [ ] Убираю лишнее: нет элементов, которые только «украшают» и не помогают решать задачу
- [ ] Информация считывается быстро: заголовки, цифры, действия на своих местах
- [ ] Интерфейс не отвлекает: нет лишней анимации, мигания, перегруженных карточек
- [ ] Предприниматель может работать с интерфейсом ежедневно без усталости от «шума»
